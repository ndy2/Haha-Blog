    What is Gradle?¶

    • build automation tool
      • takes your code and packages it into deployable unit
      • applies to small or large project
    • written in the Groovy language
    • highly configurable

    Why Use Gradle?¶

    • makes building & running easy
    • no need to install
      • Gradle wrapper comes bundled with project
      • just ./gradlew build
    • very concise

    Key Concepts¶

    1. build.gradle¶

    • build.gradle is the Bradle build script file
      • written in a Groovy DSL
      • = Maven's pom.xml
      • nomarlly lives in top level of the project
    super simple build.gradle
    puglins { // (1)!
    id 'java' // (2)!
}

group 'org.example'  // (3)!
version '1.0-SNAPSHOT'

repositories { // (4)!
    mavenCentral()
}

dependencies { // (5)!
    testCompile group: 'junit', name: 'junit', version: '4.12'
}
    1. plugins - additional build functionality
    2. plugin java - 자바 코드를 compile, test 할 수 있도록 한다.
    3. build metadata - information about your build
    4. repositories - where to find dependencies
    5. dependencies - requirements for building your code

    2. Tasks¶

    • defines a unit of work
    • invoked from the command line
      • ./gradlew build
    • see all available tasks - ./gradlew tasks
    • can create your own custom tasks
    • tasks have dependencies on other tasks
    • creates a tasks graph

    3. Wrapper¶

    • script used to invoke Gradle and run tasks
      • 🍎./gradlew build
      • Win gradlew.bat build
    • git 등의 version control 에 항상 포함해야한다.

    Simple Gradle Project Structure¶

    .
├── build.gradle
├── gradle
│   └─wrapper
|      └──gradle-wrapper.jar
|      └──gradle-wrapper.properties
|── gradlew
|── gradlew.bat
└── settings.gradle

    위의 모든 파일은 git 에 포함되어야 한다. .gradle 은 포함되면 안된다.
