### 읽기 전

호주에 거주 중이신 토비님이 SNS에 이 책을 읽고 공부하고 싶은데 책을 읽을 방법이 없다고 한탄하는 글을 게시하셨다.

![[images/toby-sns.png|400]]

끝없이 공부를 하고 커뮤니티의 리더로서 책임(?) 을 다하는 모습이 참 멋지다고 생각한다. 동시에 해당 책에 대한 궁금증이 생겨 목차를 읽고 나니 사지 않을 수가 없었다.

구매하고 간단히 읽어보니 아는 내용과 모르는 내용이 적절하게 섞여 있다는 느낌을 받았다.

책을 읽고 학습하기 좋은 신호라고 생각한다.

### 읽은 후

지금껏 스프링 관련하여 학습해온 모든 내용을 한권의 책으로 복습할 수 있어서 정말 좋았습니다. 마이크로 서비스 아키텍쳐 내에서 한 컴포넌트로서 동작할 스프링 애플리케이션을 개발하는데 필요한 지식을 압축시켜놓은것 같았습니다. 잘 알려진 내용 뿐만 아니라 저자분이 개발을 하며 겪은 다양한 상황과 생각들을 이야기 해주는 점도 참 좋았습니다.

<br>
다만 이 책한권으로 MSA 와 마이크로 서비스 스프링 애플리케이션 개발에 대한 모든 지식을 얻을 수는 없다고 생각합니다. 이 책은 한 컴포넌트라는 좁은 관점의 MSA 의 요소에 대해 집중하였습니다. MSA의 전체의 구성과 그들의 유기적인 연결 그리고 그것을 도와주는 Spring Cloud 에 대한 내용은 다른 책 혹은 실습을 통해 이루어 져야 할것 같습니다.
